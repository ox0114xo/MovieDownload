🚀 全能必殺技：智能分享攔截腳本 (Python 防彈版)

為了解決安卓系統背景分享時容易發生 Permission denied 權限報錯的問題，我們捨棄傳統做法，直接寫入純 Python 引擎。複製貼上這段「終極代碼」，完全免疫權限報錯，一鍵通殺純網址與混合圖文分享！

💡 秘訣：在各大 App 點擊「分享給 Termux」，若跳出黑色詢問視窗，請點擊右下角 EDIT 即可啟動引擎！

# 1. 強制清除所有卡死的舊檔案
rm -f ~/bin/termux-url-opener
rm -f ~/bin/termux-file-editor
mkdir -p ~/bin

# 2. 寫入無敵的 Python 攔截引擎
cat << 'EOF' > ~/bin/termux-url-opener
#!/data/data/com.termux/files/usr/bin/python
import sys, os, re

print("\n🚀 正在啟動純淨下載引擎...")
try:
    content = sys.argv[1]
    
    # 自動判斷是純網址還是圖文混合
    if os.path.isfile(content):
        with open(content, 'r', encoding='utf-8') as f:
            content = f.read()
            
    # 精準捕捉網址
    urls = re.findall(r'https?://[a-zA-Z0-9./?=_%:-]+', content)
    if urls:
        url = urls[0]
        print(f"🔗 鎖定目標網址: {url}\n")
        # 底層直連，免疫權限與路徑遺失問題
        os.system(f'"{sys.executable}" -m yt_dlp "{url}"')
    else:
        print("❌ 找不到有效網址！")
except Exception as e:
    print(f"❌ 發生致命錯誤: {e}")

input("\n✅ 任務結束！請按 Enter 鍵關閉視窗...")
EOF

# 3. 賦予權限並共用引擎
chmod +x ~/bin/termux-url-opener
cp ~/bin/termux-url-opener ~/bin/termux-file-editor
✅ 程式碼已複製！
